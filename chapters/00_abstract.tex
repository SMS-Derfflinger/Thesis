\MakeAbstract{
    操作系统教学需要兼顾经典机制讲解与现代系统软件实践。Unix V6++ 是同济大学长期使用的操作系统教学平台，其结构清晰、代码规模适中，适合展示进程管理、内存管理、文件系统和设备管理等核心机制。但原系统基于 IA32 架构和 C++ 实现，存在体系结构较旧、内存安全依赖人工保证、模块边界不够清晰等问题，难以充分体现当前系统编程领域的发展趋势。

    本课题围绕 Unix V6++ 的现代化改造展开，首先分析原系统内存管理、文件系统、缓冲区缓存和设备管理模块的实现方式与局限；随后使用 Rust 对关键模块进行重构，在保留原有教学结构和 Unix V6 风格设计的基础上，引入类型化地址抽象、Buddy 与 Slab 分层内存分配、引用封装、生命周期管理和显式错误处理等机制；最后探索向 RISC-V 64 位架构迁移，适配分页、串口、块设备和 QEMU 虚拟化运行环境。

    测试结果表明，重构后的系统能够保持基本功能正确性，在文件系统与内存管理等核心路径上具有更清晰的模块组织和更好的可维护性。Rust 与 RISC-V 的引入在保留 Unix V6++ 教学价值的同时，为教学操作系统提供了更符合现代体系结构和系统编程实践的发展方向。
}{Unix V6++，Rust，RISC-V，内存管理，文件系统，操作系统教学}

\MakeAbstractEng{
    Operating system education needs to combine classical kernel mechanisms with modern system programming practice. Unix V6++ is a teaching operating system platform used at Tongji University. Its small code base and clear structure make it suitable for demonstrating process management, memory management, file systems, and device management. However, the original system is implemented in C++ on the IA32 architecture, which leads to limitations such as outdated architecture assumptions, manually maintained memory safety, and unclear module boundaries.

    This thesis studies the modernization of Unix V6++. It first analyzes the design and limitations of the original memory management, file system, buffer cache, and device management modules. Then it refactors key modules in Rust while preserving the teaching-oriented structure and Unix V6 style design. The refactoring introduces typed address abstractions, a Buddy and Slab based layered allocator, reference wrappers, lifecycle management, and explicit error handling. Finally, the system is partially ported to the RISC-V 64-bit architecture by adapting paging, serial devices, block devices, and the QEMU virtualized environment.

    The test results show that the refactored system preserves basic functional correctness and provides clearer module organization and better maintainability in core paths such as file system operations and memory management. By introducing Rust and RISC-V while retaining the teaching value of Unix V6++, this work provides a practical direction for the evolution of teaching operating systems.
}{Unix V6++, Rust, RISC-V, memory management, file system, operating system education}
