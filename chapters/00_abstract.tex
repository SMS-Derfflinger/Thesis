\MakeAbstract{
    Unix V6++ 是同济大学操作系统课程使用的教学内核，原系统面向 IA32 架构并采用 C++ 实现。为降低内存管理和文件系统中裸指针、全局状态和手工资源约定带来的维护成本，本文围绕 Rust 重构与 RISC-V 迁移开展工作。

    首先，针对页对象语义不清、小对象分配粒度较粗和释放责任分散的问题，本文重构物理页、页级分配、内核堆和交换区管理，引入类型化地址、Buddy 与 Slab 分层分配以及 RAII 资源回收。其次，针对 inode、文件对象、文件描述符和缓冲区缓存之间引用关系分散的问题，本文保留超级块、目录项和多级块映射，整理对象持有关系和错误传播路径。最后，针对 RISC-V 迁移中的底层依赖变化，本文适配 Sv39 分页、MMIO UART 和 VirtIO 块设备，使文件系统上层逻辑能够跨平台复用。

    测试结果表明，Rust IA32 和 Rust RISC-V 版本均能完成文件系统装载、shell 运行、文件读写和用户程序执行；系统调用、动态内存分配和文件 I/O 测试未显示不可接受的整体开销。本文工作使资源生命周期、错误处理和平台边界表达更加明确。
}{Unix V6++，Rust，RISC-V，内存管理，文件系统，教学操作系统}

\MakeAbstractEng{
    Unix V6++ is a teaching operating system kernel used in the operating system course at Tongji University. The original system targets IA32 and is implemented in C++. To reduce the maintenance cost caused by raw pointers, global states, and manually maintained resource conventions in memory management and the file system, this thesis studies the Rust refactoring and RISC-V porting of Unix V6++.

    First, to address unclear page semantics, coarse-grained allocation, and scattered release responsibilities, this thesis refactors physical pages, page allocation, kernel heap management, and swap-space management, and introduces typed addresses, a layered Buddy and Slab allocator, and RAII-style reclamation. Second, to address scattered references among inodes, file objects, file descriptors, and buffer cache entries, this thesis reorganizes ownership and error propagation while preserving the superblock, directory entries, and multi-level block mapping. Third, to support RISC-V migration, this thesis adapts Sv39 paging, MMIO UART, and VirtIO block device paths, enabling cross-platform reuse of the upper file-system logic.

    Tests show that the Rust IA32 and Rust RISC-V versions can mount the file system, run the shell, perform file operations, and execute user programs. System call, dynamic memory allocation, and file I/O tests do not show unacceptable overall overhead. This work makes resource lifetimes, error handling, and platform boundaries more explicit.
}{Unix V6++, Rust, RISC-V, memory management, file system, teaching operating system}
