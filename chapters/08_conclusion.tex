
\chapter{总结与不足}

\section{全文工作总结}

本课题以同济大学 Unix V6++ 操作系统教学平台为对象，围绕内存管理和文件系统两个核心模块展开 Rust 重构，并完成了面向 RISC-V 64 位平台的底层适配与迁移。

课题首先对原系统的结构与实现方式进行了系统分析。原系统在 IA32 架构上以 C++ 实现，覆盖进程管理、内存管理、文件系统和设备管理等基本模块，具有良好的教学可读性，但在线程安全边界、资源生命周期管理和体系结构适配方面积累了一定维护成本。IA32 特有的分段与分页并存机制、传统 ATA 设备接口以及 C++ 底层大量依赖裸指针和手工引用计数的写法，使系统在向新平台迁移和向后续课程扩展时面临较多障碍。

在内存管理模块的重构中，Rust 版本引入了 Buddy 与 Slab 分层分配结构，将页级资源管理和对象级小块分配分离，改善了原系统页级分配粒度较粗的问题。物理页以统一的 PhysPage 结构表示，页框号、物理地址和地址范围通过类型化抽象加以区分。交换区管理以显式空闲区间表建模，分配、释放和合并逻辑集中在独立模块中，取代了原有分散于过程式代码中的空间管理方式。页表与地址空间管理也完成了从 C++ 位域操作到 Rust bitflags 封装的迁移，页表项语义更为明确。

在文件系统模块的重构中，Rust 版本保留了超级块、inode 缓存、目录项查找、打开文件表和多级块映射等 Unix V6 风格机制，重点重新组织了内存 inode、文件对象和文件描述符表的引用关系与生命周期管理路径。InodeRef、InodeRefGuard 和 FileRef 等包装类型替代了原系统中依赖人工约定维持的裸指针共享方式；Result<T, E> 和 PosixError 枚举组织了内部错误传播与系统调用边界的映射，使错误路径更易追踪。

缓冲区缓存与设备管理模块的重构在保留原有块缓存算法和 ATA DMA 驱动模型的基础上，以更明确的数据结构封装缓冲区状态，并引入侵入式链表适配器统一管理空闲队列、设备缓存队列和 I/O 请求队列。块设备和字符设备分别抽象为 Rust trait，传统设备开关表思想在新的类型系统中得到保留。

在 RISC-V 架构迁移方面，课题围绕启动环境、Sv39 分页、UART 串口和 VirtIO 块设备完成了从 IA32 平台到 RISC-V 64 位平台的底层支撑重建。Sv39 三级页表的页目录项和页表项以 Rust 结构体加 bitflags 封装表示；串口驱动基于 MMIO UART 实现，不依赖 SBI 字符输出；块设备驱动基于 virtio-drivers 库完成初始化和数据传输。文件系统上层结构在迁移后基本保持不变，平台相关变化主要被限制在页表、trap 机制和设备驱动等底层模块中。

最后，课题通过功能测试和性能测试对三个版本进行了比较。功能测试表明，Rust + IA32 和 Rust + RISC-V 两个版本均能完成文件系统装载、shell 运行、文件读写和用户程序执行等基本任务。性能测试从系统调用延迟、内存分配开销和文件 I/O 吞吐量三个角度进行对比，结果显示重构版本具备可接受的运行开销，若干文件系统操作的系统调用延迟在重构后甚至有所降低。

\section{主要成果归纳}

本课题的主要成果可归纳为以下几个方面。

其一，完成了 Unix V6++ 内存管理模块的 Rust 重构，建立了以 Buddy 与 Slab 协同工作为核心的分层内存分配机制，引入类型化地址抽象和 RAII 风格资源回收，使物理页、内核堆和交换区的管理边界更加明确，为内核对象的安全分配与释放提供了结构基础。

其二，完成了 Unix V6++ 文件系统模块的 Rust 重构，在保留超级块、inode、目录项和缓冲区缓存等 Unix V6 风格机制的前提下，通过引用封装和自动析构机制整理了内存 inode 与文件对象的生命周期路径，通过 Result 和 PosixError 统一了内部错误传播与系统调用接口，提升了文件系统模块的可维护性。

其三，完成了缓冲区缓存与设备管理模块的 Rust 重构，以侵入式链表和 trait 抽象替代了原系统中直接操作链表指针和函数指针数组的方式，使缓冲区状态管理和设备接口组织更为规范，为文件系统与设备层的协作提供了更稳定的接口基础。

其四，完成了面向 RISC-V 64 位平台的系统迁移，建立了新的启动环境、Sv39 分页机制和虚拟地址空间布局，适配了 MMIO UART 串口驱动和 VirtIO 块设备驱动，验证了文件系统装载、控制台初始化和 shell 运行等主要路径，证明原系统的教学结构能够在新的体系结构上继续工作。

其五，通过三版本对比测试验证了重构与迁移的可行性。测试结果表明，Rust 版本在对象生命周期、错误返回和模块边界方面的表达更明确，且没有引入不可接受的运行开销，为 Unix V6++ 的后续教学使用和进一步扩展提供了一套结构更清晰的实现基础。

\section{存在的不足}

尽管本课题完成了内存管理、文件系统和 RISC-V 迁移的主要目标，但在实现深度、测试覆盖和设计完善程度上仍存在一定不足。

在功能覆盖方面，本课题重构的范围集中于内存管理和文件系统模块，进程管理中的调度逻辑、信号机制和 fork/exec 路径并未纳入 Rust 重构范围。原系统的进程调度仍依赖 C++ 实现，与 Rust 模块之间存在语言边界，尚未形成完全统一的实现风格。这在一定程度上制约了模块间接口的一致性，也使跨模块资源关系的静态检查存在盲区。

在内存管理方面，当前交换区实现采用首次适配的空闲区间管理策略，算法较为简单，在交换区使用较为频繁时可能产生碎片。Buddy 分配器的最大阶数和 Slab 分配器的槽位大小范围目前以静态常量确定，缺少运行时调整的机制，对于教学场景中探索不同分配策略的实验支持有限。

在文件系统方面，当前实现保留了 Unix V6 风格的固定大小目录项和有限的 inode 地址表，文件大小上限受二次间接块层数约束，暂不支持更大文件或可扩展目录格式。此外，文件系统目前仅支持单一挂载点，多文件系统挂载和挂载点切换路径尚未实现，限制了其在更复杂实验场景下的适用性。

在设备与驱动方面，RISC-V 版本的 VirtIO 块设备驱动当前侧重功能正确性，未引入请求合并和异步 I/O 路径优化，导致在较大文件 I/O 测试中吞吐量低于 IA32 版本。PLIC 外部中断控制器的初始化和多中断源管理也相对简化，部分中断场景依赖轮询完成，与完整的中断驱动 I/O 路径仍有差距。

在测试方面，当前性能测试依赖 60Hz 系统时钟 tick 进行微秒级估算，计时精度有限，测试结果的绝对数值参考意义受到约束。功能测试覆盖了基本文件操作和 shell 运行路径，但针对边界条件、并发访问和错误恢复路径的系统化测试尚不完整，Rust 版本引入的引用封装和生命周期约束在实际运行中的覆盖程度有待进一步验证。

在安全性方面，尽管 Rust 重构收紧了大部分资源管理边界，底层页表操作、MMIO 寄存器访问和进程上下文切换等路径仍包含较多 unsafe 代码。这些 unsafe 边界虽已局限在较小范围内，但目前缺乏系统化的 unsafe 边界文档和审查记录，对于将系统用于教学安全性分析的场景存在一定局限。

\section{后续工作展望}

针对上述不足，后续工作可从以下几个方向继续推进。

在模块覆盖方面，可将进程管理中的 fork、exec、wait 和调度逻辑逐步纳入 Rust 重构范围，消除与 C++ 实现之间的语言边界，使系统形成更统一的模块接口风格，并进一步发挥 Rust 类型系统在跨模块资源关系检查方面的优势。

在内存管理方面，可为 Slab 分配器引入多尺寸对象缓存和每类对象独立的 Slab 链表，改善小对象分配的局部性；交换区管理可引入更完善的碎片合并策略，或替换为位图式空闲页管理，以提升交换区利用率。在 RISC-V 版本中，还可探索大页映射支持，以减少内核空间映射的页表层数。

在文件系统方面，可在保留 Unix V6 教学结构的前提下扩展 inode 地址表，支持三次间接块访问以适应更大文件；目录实现可引入可变长度目录项或哈希目录，改善路径查找性能。多挂载点支持和只读文件系统挂载也是增强系统在复杂实验场景下适用性的自然方向。

在设备与驱动方面，VirtIO 块设备驱动可引入请求合并机制和完整的中断驱动 I/O 路径，提升大文件读写的吞吐量。PLIC 初始化可进一步完善多中断源配置，使串口、块设备和时钟中断能够通过统一的外部中断路径处理，减少轮询依赖。

在测试与验证方面，可引入更高精度的计时机制，例如利用 RISC-V 的 cycle 计数器或 QEMU 提供的硬件性能监测接口，以获取更可靠的性能数据。系统化的边界条件测试和错误路径测试，以及对 unsafe 边界的文档化审查，也是提升系统教学质量和工程可信度的重要方向。

在教学适配方面，后续可结合课程实验需求，为重构后的系统补充面向学生的模块说明文档和实验接口，使 Rust 版本的内存管理和文件系统能够更自然地融入操作系统课程的实验体系，为学生理解资源管理、错误处理和体系结构适配提供更直接的观察对象。
