\chapter{测试与结果分析}

\section{测试方案设计}

本章验证 Rust 重构和 RISC-V 迁移后的功能正确性，并比较不同版本在典型用户态操作中的性能表现。测试对象包括原始 C++ + IA32 版本、Rust + IA32 重构版本和 Rust + RISC-V 迁移版本。三者均在 QEMU 虚拟环境中运行，以减少物理硬件差异对测试结果的影响。

测试环境统一控制以下条件：

\begin{enumerate}
  \item QEMU 版本；
  \item 宿主机硬件与操作系统环境；
  \item 虚拟机分配内存大小；
  \item 编译优化选项与编译器版本。
\end{enumerate}

性能测试使用三个用户态测试程序完成。系统调用延迟测试程序 \texttt{syscall\_bench.c} 循环执行 open/close、read、seek、stat 和 write 等操作，使用 \texttt{times()} 记录前后时钟并估算平均耗时。内存分配测试程序 \texttt{malloc\_bench.c} 对不同大小的内存块重复执行 malloc/free，用于观察用户态动态分配接口的基本开销。文件 I/O 吞吐量测试程序 \texttt{io\_throughput.c} 反复读取源文件并写入目标文件，统计总传输字节数和估算吞吐率。

测试程序使用系统时钟 tick 估算微秒级时间，系统时钟频率为 60Hz。因此，测试结果主要用于观察数量级和版本间相对差异，不作为精确硬件计时结果。原始 C++ 版本暂不支持 malloc 测试所需的用户态动态分配接口，内存分配性能只比较 Rust + IA32 与 Rust + RISC-V 两个版本。

\section{功能测试}

功能测试用于验证重构后的文件系统、内存管理和基础设备路径是否能够稳定工作。测试内容包括页分配与用户空间增长、文件创建与删除、文件打开与关闭、文件读写、目录创建与路径解析、控制台输入输出以及 shell 长时间运行后的基本稳定性。测试过程中使用 ls、cat、mkdir、forks 等已有用户程序，并结合新增测试程序进行命令验证。

功能测试结果显示，Rust + IA32 版本能够完成原教学系统中的基本文件操作和用户程序执行流程；Rust + RISC-V 版本能够完成串口初始化、文件系统装载、shell 运行和块设备读写等关键路径。这说明内存管理、文件系统和设备访问路径已经在两个 Rust 版本中形成可运行闭环。

\section{结果分析}

系统调用延迟结果显示，Rust + IA32 版本在 open/close、read 和 stat 等文件系统相关操作上低于原始 C++ + IA32 版本。这一差异不能简单归因于语言本身，更可能与重构过程中对文件对象、inode 引用和错误处理路径的整理有关。Rust 版本中，文件对象和 inode 的生命周期通过引用封装和析构逻辑集中管理，路径解析、文件表分配和资源回收的职责边界更清楚。write 操作中三者差异较小，说明写路径更多受缓冲区缓存、块设备写回和 QEMU 虚拟设备行为影响，而不完全取决于上层文件对象管理方式。

Rust + RISC-V 版本在 read、seek、stat 等操作上与 Rust + IA32 接近，open/close 稍慢，write 与原始 C++ 版本接近。考虑到 RISC-V 版本底层已经切换为 Sv39 页表、trap 机制和 VirtIO MMIO 块设备，该结果说明文件系统上层结构在迁移后基本保持稳定，主要差异集中在底层设备和体系结构适配路径。换言之，文件系统核心算法可以在不同体系结构上复用，平台相关变化主要被限制在页表、trap 和设备驱动等底层模块中。

内存分配测试中，Rust + RISC-V 在 64B 至 1KB 的小对象分配上低于 Rust + IA32，而在 2KB 时二者接近，在 4KB 时 RISC-V 版本开销更高。这与分配器的分层设计基本一致：小对象通常由更细粒度的堆分配路径处理，能够复用已获得的内存区域；当分配大小接近或达到页大小时，测试更容易触及页级分配、映射或空闲区管理路径，开销随之上升。原始 C++ 版本未纳入该项测试。就结构而言，Rust 重构通过 RAII、引用封装和生命周期约束明确表达分配对象的申请与释放关系，性能测试则说明这种资源管理方式能够满足基本动态分配需求。

文件 I/O 吞吐量结果波动较大。小文件测试中，原始 C++ + IA32 平均吞吐量较高，Rust + RISC-V 与 Rust + IA32 接近；较大位图文件测试中，Rust + IA32 表现较高，Rust + RISC-V 低于 IA32 两个版本。该结果与测试环境有关：I/O 吞吐量不仅受文件系统代码影响，也受 QEMU 设备模型、缓存命中、块设备驱动、写回策略和 60Hz tick 计时粒度影响。RISC-V 版本采用 VirtIO MMIO 块设备，当前实现更侧重正确性和结构清晰，后续仍有优化请求合并、缓存策略和异步 I/O 路径的空间。

综合来看，Rust 重构版本在保持 Unix V6++ 教学结构的同时，使内存管理和文件系统的资源关系更明确。系统调用和内存分配测试表明，类型化接口、RAII 风格封装和显式错误传播没有造成不可接受的运行开销；在若干文件系统操作上，重构后的 IA32 版本还表现出更低延迟。RISC-V 迁移版本能够在新的体系结构和设备模型上运行主要功能，其性能差异主要来自底层平台适配和驱动实现成熟度。

\section{本章小结}

本章对原始 C++ + IA32、Rust + IA32 和 Rust + RISC-V 三个版本进行了功能与性能测试。功能测试表明，两个 Rust 版本均能够完成文件系统装载、shell 运行、文件读写和用户程序执行等基本任务。性能测试从系统调用延迟、内存分配开销和文件 I/O 吞吐量三个角度进行对比。结果显示，Rust + IA32 在多项文件系统系统调用上低于原始 C++ + IA32 的延迟，Rust + RISC-V 在完成体系结构迁移后保持了可接受的系统调用表现；内存分配测试体现出分层分配路径在不同块大小下的开销变化；I/O 吞吐量受设备模型和计时粒度影响较大。总体上，测试结果支持本课题内存管理和文件系统 Rust 重构的可行性，也说明 RISC-V 迁移已经具备基本运行和进一步优化的基础。
