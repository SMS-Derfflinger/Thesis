\section{课题背景}

操作系统课程是计算机相关专业的核心课程，但传统教学往往面临缺少实践教学的困境。学生虽然能够记住各种概念和算法，却难以真正理解操作系统内部各模块如何协同工作\cite{you2000unix}。为解决这一问题，同济大学陈闳中教授带领多届研究生团队开发了Unix V6++（Unix V6++ Tongji's Edition for Education）操作系统教学平台。该系统基于经典的 Unix Version 6，使用 C++在 IA32 架构上重新实现，整个内核仅 8000 多行代码，"麻雀虽小，五脏俱全"，适合用于教学\cite{chen2013unixv6pp}。十多年来，该系统在同济大学操作系统课程中发挥了重要作用，帮助学生深入理解进程调度、内存管理、文件系统等核心机制。随着技术发展，Unix V6++面临着新的挑战。首先，IA32 架构逐渐被淘汰，现代计算机多采用 x86-64 或 ARM 架构，学生在老旧架构上学习与实际应用脱节。其次，C++代码虽然引入了面向对象特性，但仍然存在内存安全隐患，缓冲区溢出、悬垂指针等问题在系统编程中难以完全避免。第三，虽然 2024 年部分同学进行了现代化改造（如支持 QEMU、优化编译系统），但核心架构仍保持 2008 年的设计，缺乏对现代编程语言和指令集架构的支持。与此同时，系统编程领域出现了两个重要趋势。一是 RISC-V 开放指令集架构的兴起。RISC-V 由加州大学伯克利分校于 2010 年提出，采用完全开源的 BSD 许可证，指令集设计简洁规范，模块化扩展机制使其在教学和工业界迅速普及。Waterman 和Asanović 在《The RISC-V Reader》中介绍，RISC-V 的精简设计特别适合操作系统教学，学生可以在几周内掌握完整的指令集\cite{waterman2017riscv}。二是 Rust 系统编程语言的成熟。Rust于 2015 年发布 1.0 版本，通过所有权系统和借用检查机制在编译期保证内存安全，同时保持零成本抽象，性能媲美 C/C++\cite{klabnik2022rust}。在操作系统教学领域，已有多个基于现代技术的实践项目。MIT 的 xv6 是最著名的 Unix V6 教学重写版本，最初用 C 语言实现于 x86 架构，2019 年移植到 RISC-V\cite{kaashoek2021xv6}。清华大学陈渝团队开发的 rCore 使用 Rust 从零开始构建操作系统，配套详细的中文教程，已在多所高校推广\cite{chen2023rcore}。国外的博客项目"Writing an OS in Rust"提供了丰富的 Rust

操作系统开发经验\cite{oppermannWritingOS}。在工业界，蚂蚁集团推出的星绽操作系统也采用了 Rust 技术栈，证明了 Rust 在生产环境中的可靠性\cite{antgroup2023asterinas}。这些项目证明了 Rust+RISC-V 组合在

操作系统教学和工业应用中的可行性。基于上述背景，将 Unix V6++移植到 RISC-V 架构并使用 Rust 重写核心模块，既能保持原系统的教学价值，又能引入现代技术优势，对操作系统教学具有重要意义。

\section{研究目标}

本课题将使用 Rust 语言重新实现 Unix V6++操作系统的内存管理模块和文件系统模块，保持原有的设计思想和算法，确保教学连贯性，对比分析 Rust 版本与 C++版本在代码安全性、可维护性和性能方面的差异。再将其移植到 RISC-V 64 位架构，适配 RISC-V Sv39 分页机制等核心机制。设计合理的内核与用户态虚拟内存布局，确保地址空间隔离和系统稳定性。实现适配 RISC-V 平台的设备驱动，包括 UART 串口、VirtIO 块设备等，使系统能够在 QEMU RISC-V 虚拟机上稳定运行。并在移植过程中对部分模块进行优化改进。完成技术报告，对比分析 Rust 与 C++、RISC-V 与 IA32在操作系统开发中的异同，总结经验教训，为后续教学和研究提供参考。
